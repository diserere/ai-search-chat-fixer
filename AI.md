# AI Context & Architecture Notes

Этот файл содержит технический контекст архитектуры проекта для ИИ-ассистентов (LLM), помогающих в разработке репозитория.

## Архитектурные принципы

1. **Vanilla JS Only**: Никаких внешних библиотек (jQuery, Lodash и т.д.). Только чистый JavaScript для максимальной производительности и безопасности.
2. **Phase: Capture (`true`)**: Все перехваты клавиатуры (`keydown`) должны инициализироваться на этапе погружения событий (`window.addEventListener('keydown', handle, true)`). Это позволяет заблокировать событие `Enter` до того, как его поймают внутренние скрипты сайтов.
3. **Изоляция DOM**: Скрипты должны быть точечными. Селекторы элементов должны использовать максимально стабильные атрибуты (`name="search"`, `role="button"`), игнорируя динамически обфусцированные CSS-классы сборщиков (Webpack/Vite).

## Специфика реализации платформ

### Google AI Search
* Интерфейс использует кастомный редактируемый контейнер (не стандартный textarea).
* Перевод строки требует ручной инъекции тегов переноса.
* Метод отправки завязан на системный триггер формы.

### DeepSeek Chat
* Интерфейс построен на базе стандартного тега `<textarea>`, обернутого в кастомные UI-компоненты (кнопка отправки является элементом `div` с `role="button"`).
* **Проблема скролла и реактивности**: Прямое изменение `textarea.value` ломает Virtual DOM сайта. Решено через использование `document.execCommand('insertText', false, '\n')`, что сохраняет скроллинг и автовысоту.
* **Проблема кастомной кнопки**: Кнопка-див не реагирует на стандартный метод `.click()`. Решено через генерацию и диспетчеризацию нативного события обычного нажатия клавиши `Enter` на самом поле ввода:
  ```javascript
  const nativeEnterEvent = new KeyboardEvent('keydown', {
      key: 'Enter',
      code: 'Enter',
      bubbles: true,
      cancelable: true
  });
  target.dispatchEvent(nativeEnterEvent);
  ```

## Стандарты ветвления и коммитов
* Проект строго следует **Trunk-Based Development** (все фиксы и фичи сразу вливаются в main-ветку после тестирования).
* Сообщения коммитов должны строго соответствовать спецификации **Conventional Commits**.

### Спринт 2: Интеграция DeepSeek & Мультиплатформенность
* **Архитектурный сдвиг**: Проект эволюционировал из фиксера одного чата в экосистему `ai-search-chat-fixer`. Файлы разделены по префиксам в `src/`.
* **Критический инсайт (Внимание ИИ!)**: Современные SPA-интерфейсы (React/Vue) защищены от прямой модификации DOM (`.value`) и стандартных триггеров (`.click()`). Попытки манипулировать ими «в лоб» ломают Virtual DOM (теряется скролл, блокируются кастомные кнопки-дивы).
* **Принятый стандарт**: Взаимодействие с реактивными чатами реализуется строго через **эмуляцию пользовательского ввода** низкого уровня:
  * Перенос строки: `document.execCommand('insertText', false, '\n')` (сохраняет нативный скролл фреймворка).
  * Отправка: генерация чистого `KeyboardEvent('keydown', { key: 'Enter' })` в поле ввода (заставляет движок сайта выполнить отправку самостоятельно).
* **Вектор на будущее**: При подключении новых платформ (Claude, ChatGPT) сначала определять тип контейнера (`textarea` vs `contenteditable`) и проверять реакцию на генерацию событий, избегая усложнения DOM-дерева.
