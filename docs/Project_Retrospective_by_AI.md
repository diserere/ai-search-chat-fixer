# Спринт 2: Интеграция DeepSeek Chat Fixer (v1.4.0) (модель: Googgle AI Search chat)

## Техническая ретроспектива

### 1. Выявленные проблемы и контекст платформы
* **Кастомные элементы управления**: Кнопка отправки сообщений в DeepSeek реализована не через стандартный тег `<button>`, а через кастомный `div` с атрибутом `role="button"`. Стандартный метод `.click()` на этапе интеграции игнорировался фреймворком.
* **Нарушение скролла и реактивности**: Попытка прямой перезаписи свойства `textarea.value` приводила к рассинхронизации с Virtual DOM (React/Vue), из-за чего ломался автоматический расчет высоты поля ввода и прекращался вертикальный скролл.

### 2. Технические решения (Архитектурные сдвиги)
* **Эмуляция KeyboardEvent вместо DOM-кликов**: Вместо поиска графической кнопки и попыток вызвать на ней событие нажатия, реализована генерация нативного системного события `KeyboardEvent('keydown')` со значением `key: 'Enter'`. Событие отправляется непосредственно в фокусное поле `textarea`. Внутренний движок сайта перехватывает его, запуская штатный механизм валидации и отправки формы.
* **Использование `document.execCommand`**: Для безопасного переноса строки без разрушения Virtual DOM задействован метод `document.execCommand('insertText', false, '\n')`. Это заставляет фреймворк сайта думать, что символ был введен пользователем вручную, сохраняя нативный автоскролл и динамическую высоту элемента.

### 3. Извлеченные уроки (Lessons Learned) для ИИ-ассистентов
* **Приоритет имитации ввода над манипуляцией DOM**: При работе с современными реактивными фреймворками (React, Vue, Svelte) имитация действий пользователя через низкоуровневые события (`KeyboardEvent`, `execCommand`) гораздо надежнее, чем прямая модификация свойств элементов (`.value = ...`) или вызовы методов `.click()`.
* **Масштабируемость структуры**: Успешно протестирован подход префиксов файлов (`google_*`, `deepseek_*`) в единой директории `src/` без усложнения дерева проекта избыточными подпапками.

## 📝 Ретроспектива AI-Driven SDLC: Спринт по интеграции DeepSeek

### 1. Ход работы (Краткий таймлайн)

* Итерация 1 (Гипотеза): ИИ предложил стандартное решение с прямой перезаписью .value и поиском тега button.
* Тестирование 1 (Сбой): Пользователь применил строгую буквенную методологию логирования (s, a, e, c), которая мгновенно подсветила две проблемы: потерю реактивного скролла и отказ кнопки отправки.

* Итерация 3 (Финальное решение): Вместо продолжения борьбы с DOM, ИИ предложил перейти на уровень ниже — имитировать действия пользователя через KeyboardEvent и execCommand. Тестовый лог пользователя подтвердил 100% успех.

### 4. Впечатления и пожелания ассистента 🚀
